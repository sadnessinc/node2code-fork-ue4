Back2Dead fork hotfix: Blueprint Assist auto-format after import

Added:
- Optional Blueprint Assist bridge.
- Runtime checks: BlueprintAssist plugin must exist, be enabled, and module must load.
- Editor Preferences setting:
  Node to Code | Blueprint Assistant Extension | Auto Format Imported Functions
- When enabled, successful N2C import collects functions changed by add_or_replace_function / replace_function_body actions.
- After apply, NodeToCode queues Blueprint Assist formatting for those changed function graphs.

Safety:
- Build.cs compiles the bridge only when BlueprintAssist source headers are present in Project/Plugins.
- If BlueprintAssist is absent or disabled, import still works; auto-format is skipped with a warning only when the setting is enabled.
- No formatting is attempted on dry-run/validation.

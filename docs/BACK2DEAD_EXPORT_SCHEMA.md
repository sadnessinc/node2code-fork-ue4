# N2C_AI_EXPORT_V2 schema — Back2Dead fork

`N2C_AI_EXPORT_V2` is generated by the **Export N2C** button.

It is optimized for AI/code review and patch generation. It is not a binary-safe clone of a Blueprint asset.

---

## Root fields

```text
schema = N2C_AI_EXPORT_V2
export_kind = Blueprint | NiagaraReflection
exported_at
engine_target = UE4.27
metadata
variables
functions
warnings
```

---

## Blueprint metadata

```json
"metadata": {
  "blueprint_name": "A_Room",
  "blueprint_path": "/Game/.../A_Room.A_Room",
  "blueprint_class": "...",
  "parent_class": "...",
  "is_blueprint_dirty": false
}
```

---

## Function object

Every Blueprint function/event graph/macro/delegate graph is exported as an item in `functions[]`.

```json
{
  "name": "RoomGen_FindAdjacentGroundCellForPlatform",
  "graph_type": "function",
  "graph_class": "EdGraph",
  "nodes": [],
  "exec_edges": [],
  "data_edges": [],
  "linear_flow": []
}
```

`graph_type` values:

```text
function
event_graph
macro
delegate_signature
unknown
```

## Pin object

```json
{
  "id": "...",
  "name": "Then",
  "display_name": "Then",
  "direction": "output",
  "pin_type": {
    "category": "exec",
    "sub_category": "",
    "container": "None",
    "is_reference": false,
    "is_const": false,
    "is_weak_pointer": false
  },
  "is_exec": true,
  "connected": true,
  "links": []
}
```

Enum pins include `enum_values[]` when the enum object is available:

```json
{
  "internal_name": "NewEnumerator3",
  "display_name": "Platform"
}
```

This is important for UE enum display-name issues.

---

## Edges

Readable exec/data edges are exported separately:

```json
{
  "from_node_id": "N1",
  "from_node": "Function Entry",
  "from_pin": "Then",
  "to_node_id": "N2",
  "to_node": "Branch",
  "to_pin": "Execute",
  "readable": "Function Entry.Then -> Branch.Execute"
}
```

---

## Linear flow

`linear_flow[]` is a simple execution traversal from entry/event nodes.

It is not a full compiler IR. It exists to make review faster.

# Export ZIP update

The `Export N2C` toolbar action now saves the normal folder export and also attempts to create a shareable ZIP next to that folder:

```text
Saved/NodeToCode/AIExports/N2C_AI_<Blueprint>_<Timestamp>/
Saved/NodeToCode/AIExports/N2C_AI_<Blueprint>_<Timestamp>.zip
```

ZIP export is created natively in C++ with a simple ZIP32 writer; it does not require PowerShell, 7z, or shell extensions.
If ZIP creation fails, the folder export remains valid and the report contains a warning.
